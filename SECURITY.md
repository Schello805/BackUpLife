# Security Policy

## Unterstützte Version

Aktuell wird nur die jeweils neueste Version im Repository unterstützt.

## Meldung von Sicherheitslücken

Bitte veröffentlichen Sie Sicherheitslücken nicht direkt als öffentliches Issue.
Melden Sie sicherheitsrelevante Probleme zunächst vertraulich an den Maintainer.

## Sicherheitsfokus dieses Projekts

- Schutz gespeicherter Zugangsdaten
- saubere Rollentrennung zwischen Admin, Ersteller und Leser
- nachvollziehbare Audit-Logs
- sparsame und klare Freigabelogik
